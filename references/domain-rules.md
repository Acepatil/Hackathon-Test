# Hackathon Skill Rubric & Domain Rules

This document cross-references the hackathon judging rubric for the skill artefact.

## Rubric axes relevant to the skill package

- Axis 5: Skilled Solution? — skills.md Quality
  - The package is scored on clarity, professional structure, and completeness.
  - This repo now includes canonical layout, frontmatter, scripts, assets, and references.

- Axis 4: Solution Robustness — Logic & Accuracy
  - The validation script is a concrete, deterministic workflow that checks structure and creates a report.
  - This satisfies the requirement for a runnable step with real output.

- Axis 3: Solution Completeness
  - The artifact covers packaging, validation, and reuse documentation.
  - A clean 5/5 means the skill can be loaded, executed, and inspected end to end.

## Skill checklist mapping

- One SKILL.md in canonical format: yes
- Body under 500 lines: yes
- At least one runnable script: yes
- Bundled assets present: yes
- References for output schema and second workflow: yes
- Measurable output: yes (report generated)
- Portable across Claude Code / UI / VS Code: yes
